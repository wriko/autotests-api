Очистка проекта от учебных файлов
В ходе прохождения курса мы создавали множество тестовых файлов для изучения различных технологий. Например, в папке /tests/pytest/ находятся файлы, которые не выполняют реальные тесты, а использовались только для практики работы с pytest.

Почему эти файлы не должны быть в финальном проекте, который вы добавите в резюме:

Финальный проект должен выглядеть профессионально. Наличие учебных или временных файлов производит впечатление незавершённости.
Избыточное количество файлов делает проект громоздким и менее читаемым.
Риск упустить внимание работодателя. Работодатель не будет изучать каждый файл в проекте. Если он откроет репозиторий и увидит учебный код, это может создать впечатление неаккуратности и снизить ваши шансы на успех.


Совет:  Если вы хотите сохранить учебные файлы для повторения в будущем:

Создайте копию проекта в отдельной папке на вашем компьютере.
Создайте новый репозиторий на GitHub, добавьте туда учебные файлы и сделайте репозиторий приватным.
Так вы всегда сможете вернуться к исходному коду для повторения.
Удаляем учебные файлы:
Удалите все файлы tcp_*.py в корне проекта, которые были созданы для изучения TCP/IP протоколов.
Удалите все файлы websocket_*.py в корне проекта, которые были созданы для изучения WebSocket протокола.
Удалите все файлы grpc_*.py, *.proto, *_pb2.py, *_pb2_grpc.py в корне проекта, которые были созданы для изучения gRPC протокола.
Удалите все файлы httpx_*.py в корне проекта, предназначенные для изучения HTTPX.
Удалите все файлы pydantic_*.py в корне проекта, предназначенные для изучения Pydantic.
Удалите все файлы pydantic_json_schema_*.py, json_schema_*.py в корне проекта, предназначенные для изучения JSON Schema.
Удалите все файлы api_client_*.py в корне проекта, предназначенные для изучения API Client.
Удалите все файлы faker_*.py в корне проекта, предназначенные для изучения Faker.
Удалите все файлы json_*.py, json_*.json, xml_*.py, xml_*.xml в корне проекта, предназначенные для изучения форматов представления данных.
Удалите все файлы из папки /tests/pytest/, так как это учебные материалы по pytest.


Теперь ваш проект будет выглядеть чисто и профессионально, готовым для добавления в резюме.
